PYTHON PROGRAM – REVERSE A THREE DIGIT NUMBER
---------------------------------------------

This program reverses a three-digit number entered by the user.

no = int(input("Enter a number: "))

d1 = no % 10
no = no // 10
d2 = no % 10
d3 = no // 10

rev = d1*100 + d2*10 + d3
print("Reversed number:", rev)

Example
Input : 123
Output : 321


================================================
ALGORITHMS AND DATA STRUCTURES
================================================

Algorithm
---------
An algorithm is a step-by-step procedure used to solve a problem in a program.

Examples:
Sorting numbers
Searching data
Finding factorial of a number


Data Structures
---------------
A data structure is a method of storing and organizing data so that it can be used efficiently.

Examples:
Array
Linked List
Stack
Queue
Tree
Graph


Classification of Data Structures
---------------------------------

1. Primitive Data Structures
These are basic built-in data types.

Examples:
Integer
Float
Character
String
Boolean


2. Non-Primitive Data Structures
These structures are created using primitive data types.

They are divided into two main types.

A) Linear Data Structures
Data elements are stored sequentially.

Examples:
Array
Linked List
Stack
Queue


B) Non-Linear Data Structures
Data elements are connected in hierarchical or network form.

Examples:
Tree
Graph


================================================
BIG O NOTATION
================================================

Big O notation is used to describe the efficiency of an algorithm when the input size grows.

It helps programmers understand how the running time or memory usage changes as the amount of data increases.

Example:
Comparing how a program behaves with 10 inputs versus 1,000,000 inputs.


Types of Complexity
-------------------

1. Time Complexity
Time complexity measures how long an algorithm takes to execute.

Example:
Program A runs in 30 seconds
Program B runs in 60 seconds

Program A is more efficient.


2. Space Complexity
Space complexity measures how much memory is required during program execution.


Performance Cases
-----------------

Best Case
Minimum time required.

Average Case
Typical running time of an algorithm.

Worst Case
Maximum time the algorithm may take.


Common Big O Notations
----------------------

O(1)     Constant Time
O(n)     Linear Time
O(n²)    Quadratic Time
O(log n) Logarithmic Time
O(2ⁿ)    Exponential Time

Other Notations:
Big Omega (Ω) – Best case complexity
Big Theta (Θ) – Average case complexity


================================================
RULES FOR MEASURING COMPLEXITY
================================================

Rule 1
A single assignment statement has complexity O(1).

Example

a = 5


Rule 2
A loop running from 0 to n has complexity O(n).

Example

for i in range(n):
    print(i)


Rule 3
Nested loops increase complexity to O(n²).

Example

for i in range(n):
    for j in range(n):
        print(i, j)


Rule 4
If the variable is divided by 2 in every iteration, complexity becomes O(log n).

Example

n = 16

while n > 1:
    print(n)
    n = n // 2


Rule 5
When multiple operations occur sequentially, their complexities are added.

Example

for i in range(n):
    print(i)

for j in range(n):
    print(j)

Total Complexity = O(n)


================================================
COMMON QUESTIONS
================================================

1. Difference Between Big O and Big Omega

Big O (O)
Represents the worst-case complexity of an algorithm.

Big Omega (Ω)
Represents the best-case complexity of an algorithm.


2. Difference Between Linear and Non-Linear Data Structures

Linear Data Structures
Data is arranged sequentially.

Examples:
Array
Stack
Queue

Non-Linear Data Structures
Data is arranged hierarchically.

Examples:
Tree
Graph


3. How O(log n) Complexity Occurs

Logarithmic complexity happens when the input size reduces by half in each step.

Example

n = 64

while n > 1:
    print(n)
    n = n // 2

Sequence:
64 → 32 → 16 → 8 → 4 → 2 → 1

Number of steps ≈ log₂ n


4. Grid Example Complexity

for i in range(h):
    for j in range(w):
        print(i, j)

Time Complexity = O(h × w)
